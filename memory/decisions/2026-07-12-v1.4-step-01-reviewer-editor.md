# v1.4 Step 1 — Reviewer Editor Redesign Decisions

Requested by Alberto (2026-07-12): replace the reviewer's two-column raw editor
(textarea + preview) with a single rendered A4 view — comments in red, a
Word-style comments panel with per-reviewer private notes and click-to-jump +
highlight, comments added by selecting rendered text.

## Observations

- [decision] The reviewer editor becomes a single rendered A4 sheet (no textarea column); comments render as RED inline markers; a right panel lists them Word-style. The owner's *implement* editor is untouched (keeps editor.js + editor-grid) — two editors coexist #ui
- [decision] Model UNCHANGED: the reviewer copy is still baseline + inserted {COMM}/{SUGG} blocks. A hidden #content-src (name=content) carries the Markdown; the editor reconstructs it on submit and posts to the same POST /ui/reviews/{id}/edit-copy (freeze validation + auto-harvest). The editor now REQUIRES JS #model
- [decision] Add-comment = select rendered text -> popover (type/sev + body; SUGG prefills old=selection). Anchor by text-match: the selected string + its occurrence index counted in the rendered text before the selection, mapped to a baseline character offset; no-match falls back to end-of-baseline #add
- [decision] Rendering avoids source-offset -> DOM mapping: comment blocks are substituted into a copy of the baseline as red span markers BEFORE marked runs (marked passes inline HTML through). Only the add path maps a selection to the source #render
- [decision] Private notes: new ReviewerNote(review_id, user_id, anchor_key, body) + Alembic migration f1a2b3c4d5e6 + GET/PUT /ui/reviews/{id}/my-notes (reviewer-gated, JSON/204). anchor_key = the comment's baseline offset (stable: baseline is frozen), so a note survives edits. Scoped per reviewer; never harvested/shared #notes
- [context] Frontend: reviewer-editor.js (new) + rewritten edit_copy.html + app.css (A4 sheet, red markers, flash, panel, popover). No new runtime dep (marked vendored) #impl
- [context] Server suite green (213). The JS flows are live-verified in a browser (A4 render + red markers, card->jump+flash, note persistence, select->comment anchoring inserting the block exactly after the phrase, submit->harvest = 2 COMM RIDs). The preview screenshot tool timed out; DOM snapshots were used as proof #testing
