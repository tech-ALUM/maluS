# v1.9 Step 1 — ALUM Brand Refresh (GUI) Decisions

Requested by Alberto (2026-07-16): a redesign of the web GUI to the ALUM brand,
keeping all functions and everything that isn't design.

## Observations

- [decision] Scope = cohesive REFRESH (not a layout rework): keep structure/flows, elevate the design system (tokens, type, components, spacing) to the exact ALUM brand. Design-only — no route/service/HTMX/template-logic change, applied once in app.css + base.html so it lands on every page #scope
- [decision] Vendor the real brand fonts locally (Space Grotesk 500/600/700, Inter 400/500/600, JetBrains Mono 400/500) as @font-face woff2 under static/fonts/, honouring the no-runtime-CDN rule. Latin subset only (~264 KB total) — covers EN + Italian. Previously the GUI only NAMED the fonts and fell back to system-ui #fonts
- [decision] Color tokens aligned to the locked brand hex (alum-brand-identity references/brand.md): --paper #F7F8FA, --line #E2E7EC, --muted #5C6470; added --faint #8A929C, --surface, --coral-soft #FFF1EF, --teal-soft #E6F2F3, --teal-dark, --radius, --shadow. coral/teal/ink were already exact #tokens
- [decision] Components refreshed to the brand philosophy (coral deliberate, teal support, JetBrains Mono for ids/versions): topbar coral bottom rule + inline ALUM mark (assets/alum-mark.svg), Space Grotesk table headers + row hover, refined buttons, card/metric/panel subtle shadow, callouts on the soft tints #components
- [decision] The alum-brand-identity skill's GUIDE TEMPLATE (hero + sticky TOC) was NOT used — it targets doc pages, not a web app. Only its tokens, logo assets, and component philosophy were applied to the existing app.css. The maluS favicon/product "S" mark was kept (product identity), the ALUM mark added as a brand tie-in in the topbar #brand #boundary
- [decision] package-data for malus.web now includes static/fonts/* so the woff2 ship in the Docker image (mirrors the v1.4.2 static/vendor/* fix) #packaging
- [context] Live-verified on the login page + static assets: fonts serve (/static/fonts/inter-400.woff2 → 200 font/woff2) and the browser loads the real families, the mark serves + renders in the topbar, brand tokens present in the served CSS. The preview renderer needs an explicit viewport (defaults to innerWidth 0). No schema change / no migration. Full suite green (255) #testing
