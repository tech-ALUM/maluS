# v1.5 Step 1 — Delete a Review from the GUI Decisions

Requested by Alberto (2026-07-12): the owner should be able to delete a whole
review from the browser.

## Observations

- [decision] The primary owner (Review.owner_id) OR a global admin can hard-delete a review; reviewer/moderator cannot (403). It is irreversible (no soft-archive — YAGNI) #delete
- [decision] delete_review (services/core.py, transactional, mirrors delete_user) removes children in FK-safe order — RidChange, then RID (nulling the master_id self-ref first), ReviewerCopy, ReviewerNote, ReviewMember, DocumentVersion, Document — then the Review; writes a delete_review audit entry #delete
- [decision] AuditLog rows are KEPT (append-only; they reference the review by string target, not FK) → the trail "review X was deleted" survives #audit
- [decision] GUI: a danger "Delete review" button on the dashboard (owner/admin) -> confirm page GET /ui/reviews/{id}/delete (shows finding count + members) -> POST /ui/reviews/{id}/delete -> redirect to the list #gui
- [context] No schema change / no migration (pure deletes). Server-rendered, no JS -> covered fully by pytest (no live-browser step needed) #impl
- [context] Tests: tests/db/test_delete_review_service.py (cascade removes every dependent row) and tests/web/test_delete_review.py (owner/admin delete, reviewer/moderator 403, gone from list + 404, control hidden). The service test needed a distinct basename — pytest rejects duplicate test-module basenames without __init__.py. Full suite green (219) #testing
