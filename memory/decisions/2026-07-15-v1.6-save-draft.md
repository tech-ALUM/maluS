# v1.6 Step 1 — Reviewer Draft-Save + Submission-Status Panel Decisions

Requested by Alberto (2026-07-15): reviewers must be able to comment a bit at a
time — save a draft, see it in the RTD table immediately, and reopen their copy
another day with prior comments intact; and the dashboard should show who has
submitted, so the owner knows when to proceed.

## Observations

- [decision] Reviewer copy split into Save (draft) and Submit. `ReviewerCopy.submitted_at` (already nullable) is the signal: draft = NULL, submitted = timestamp. `ReviewerCopyRepo.upsert` no longer forces `submitted_at = now`; the policy moved to `add_reviewer_copy(..., submitted: bool = True)` (default True keeps legacy import + API submit + tests marking submitted) #editor
- [decision] Both Save and Submit validate the freeze rule server-side and re-harvest, so draft comments appear in the shared RTD table at once. Save redirects back to the editor (`?saved=1` flash); Submit redirects to the dashboard #editor
- [decision] The submission gate is a SOFT indicator (Alberto's choice over a hard block or a phase button): the dashboard panel shows each reviewer as not-started / draft / submitted with an N/M count and a "waiting for N" / "all submitted" note, but blocks nothing #gui
- [context] Accepted trade-off of the soft gate: harvest is global, so draft comments are RIDs visible to everyone; if the owner disposes a draft-sourced RID and the reviewer later edits that comment, the RID is WITHDRAWN and a new OPEN one is created (disposition lost). The note signals this; it does not prevent it #tradeoff
- [decision] API parity: `PUT /reviews/{id}/copies/{user}` saves a draft (`submitted=False`); `POST .../submit` submits (`submitted=True`). PUT keeps its no-harvest contract (the pipeline harvests separately), so no existing API caller regresses #api
- [decision] The reviewer editor form opts out of hx-boost (`hx-boost="false"`). Found by LIVE browser verification: under hx-boost (htmx 2.0.3) the submit button's `action` value is dropped, so "Save draft" fell back to the default and wrongly submitted. A native submit includes the clicked button's value and still works with JS off. Guarded by `test_editor_form_opts_out_of_hx_boost` #bug #htmx
- [context] No schema change / no migration (`submitted_at` and the panel data already exist in the model). Full suite green (234) #impl
