# Step 7 — End-to-End Validation & Release

## Objective

Prove the whole pipeline on a realistic review and package v0.1.0.

## Deliverables

- [ ] `examples/srs-demo/` — fictional SRS (~10 sections) + scripted
      3-reviewer scenario (typos, duplicates, technical majors, one freeze
      violation, one reopen) — fully synthetic content
- [ ] Scripted E2E test driving init → freeze → copies → harvest → triage →
      disposition (headless rtd.yaml edits) → implement → verify → finalize
- [ ] `docs/usage.md` — user guide for both modes (human owner / AI owner)
- [ ] README completed (install, quickstart, GIF/screenshot of GUI)
- [ ] Packaging: installable via `pipx install .`; version 0.1.0; tag `v0.1.0`

## Definition of Done

A newcomer can run the demo review start-to-finish following `docs/usage.md`
only; E2E test green in CI-less local run (`python -m pytest -q`); tagged
release pushed.
